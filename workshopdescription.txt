[h1] 过月性能优化 [/h1]

当前版本重点是为原后端建立只存在于内存中的高速缓存结构：在不延迟世界结算、不跳过原版逻辑、不写入自定义存档数据的前提下，bypass 原版过月中的低效扫描路径，并重写为基于本 Mod 高速缓存表的高效反向查询 / 候选上界查询。

这些缓存不写盘，退出世界或切档后会释放；诊断样本中主要缓存命中率普遍保持在 90% 以上，部分目标索引、matcher 和静态 planning graph 缓存命中率可突破 99%。

对密闻数量多、NPC 行动规划重、存档体积大的后期档，当前测试样本中过月计算耗时通常可压缩到原版的 50%-15% 左右，也就是约 2-8 倍的过月计算加速；具体收益取决于存档热点分布、硬盘性能和其它 Mod。

本 Mod 不接管、延迟或新增原版保存。

[h3][b]不写入自定义存档数据，不改变原版世界结算屏障；可在已有存档中自由加入/取消本mod。[/b][/h3]

[h2]优化更新#260628[/h2]
这一版继续整理 NPC 月行动规划优化，把主/副目标规划之间的缓存屏障收束为更清晰的两段式结构：

primary planning 读取冻结快照，热路径只读，尽量避免 worker 线程锁竞争。

primary ApplyAll 串行执行后记录 location / relation / inventory delta，不在 planning 热路径里抢锁更新大表。

primary ApplyAll 完成后，根据 delta 局部发布更新后的目标索引 / 关系预过滤 / 物品持有人预过滤 / matcher 依赖版本，供 secondary planning 使用；只有覆盖面过大或状态不可证明安全时才回退到更保守的路径。

这一版还补充了静态 planning graph 邻接缓存，用于减少完整新规划时的条件/效果节点扫描；对后续复用已有规划的月份，热点仍主要在 PrepareContext、FilterActionTargets 和 matcher 判断上。

诊断日志新增或细化了 delta publish、affected block / area、target lookup、target matcher、relation / item prefilter、planning graph cache 等信息，方便判断当前存档到底卡在扫描、匹配、执行、组织更新还是保存阶段。

本次更新的重点不是改变 NPC 行为，而是把高频查找路径整理成“规划阶段冻结、写回阶段记录、下一阶段局部更新”的结构。对于 NPC 行动规划占比较高的后期档，预计可以进一步降低主/副目标规划阶段的等待时间，并让之后的维护更稳。

[h2]优化更新#260627[/h2]
整理 NPC 月行动热点优化：针对 CharacterActionPlanning 中高频的目标范围扫描、关系目标、物品持有人目标和 target matcher 判断做了单独优化。

本次将 NPC 规划缓存屏障整理为统一的 OfflineUpdateCurrentGoalActions 阶段快照：目标索引、关系候选预过滤、物品持有人预过滤共用同一份冻结快照根，减少重复构建和重复世界查询。

该优化仍以严格不改变原版逻辑为目标：预过滤只缩小候选集合，最终 selector / predicate / 随机选择 / 行动执行仍走原版；target matcher 缓存只对白名单依赖和局部版本号可验证的判断生效，异常、失效或无法证明安全时回退原版路径。

预计可显著改善后期存档中 NPC 月行动阶段的等待时间，尤其是关系互动、物品目标和主/副目标规划占比较高的存档。诊断日志会输出相关命中率，便于继续确认热点。

[h2]重大更新#260626 P4[/h2]
新增 NPC Planning / CharacterActionPlanning 加速：在月结 NPC 行动规划前同步生成地块/地区/州域/聚落候选目标索引，然后在主/副目标规划阶段替换原版重复范围扫描。

上一个公开版本尚未包含 NPC planning 加速。本次更新后，密闻月结、NPC 主/副目标查找和存档 IO 三个主要热点都已纳入优化范围。

该优化严格不改变原版 NPC 决策逻辑：最终候选过滤、关系判断、随机选择、目标选择和行动执行仍走原版；本 Mod 只把“查找候选人在哪里”这一步换成预先建好的索引。

近期测试中，NPC planning 加速对 CharacterActionPlanning 占比较高的后期存档收益明显。配合密闻热路径替换，在严格不改变原版逻辑的前提下，可以进一步压缩集中在过月按钮后的等待时间。

[h2]重大更新#260626 P3[/h2]
新增 SaveWorld 存档 IO 优化：将原版 working.db 写入/读出存档时的 4KB 复制块放大到可配置档位，以减少大量小块循环带来的开销。

当前支持 1/4/8/16MB 档位，默认 4MB；可选取消存档压缩，以更大的存档体积换取更快的保存速度。

SaveWorld 诊断日志会额外输出数据库复制块大小、估算分块数、压缩收尾耗时和各 Domain.OnSaveWorld 耗时，方便比较不同设置在不同硬盘和存档体积下的实际收益。

[h2]重大更新#260626 P2[/h2]
移除跨帧延迟结算、pending 队列和 sidecar 机制，避免因延迟世界状态带来的兼容性和存档一致性风险。

优化方向改为查找并替换原版过月热路径中的重复扫描：重点优化密闻传播、持有人计数和代谢清理阶段。

新增 SaveWorld 写盘细分诊断，可判断保存耗时究竟来自 domain 序列化、working.db 复制、压缩收尾还是其他文件操作。

实验性 NPC 行动点削减保留，但默认关闭；其保护快照仍在游玩帧中按预算构建，尽量不把额外计算压回过月按钮。

[h2]重大更新#260626[/h2]
重构保护区域与缓存逻辑：当前州域始终保护，可选统一保护相邻州域；实验性行动点调整使用同一套保护判断。

新增实验性 NPC 月行动点调整：可降低未受保护远区 NPC 的每月行动点增长，以减少远区 NPC 行动循环压力；此项会改变远区 NPC 推进速度，默认关闭。

新增运行时诊断日志，方便排查不同存档中的过月热点。

目前这些优化更接近“不改变原版语义”的理想方案，也更适合作为稳定版本长期维护。
